### 1.1 排行榜展示

#### 业务规则

1. Scenario: 查看模型+Agent组合排名
- **WHEN** 用户访问排行榜页面
- **THEN** 系统展示所有已评测的模型+Agent组合，按解决率（resolved rate）降序排列，显示模型名称、Agent名称、评测数据集、解决率、总任务数、已解决任务数

2. Scenario: 按维度筛选排名
- **WHEN** 用户选择筛选条件（按模型、按Agent、按数据集）
- **THEN** 系统仅展示符合筛选条件的评测结果排名

3. Scenario: 查看特定组合的详细评测结果
- **WHEN** 用户点击某个模型+Agent组合
- **THEN** 系统展示该组合的详细评测结果，包括每个任务的通过/失败状态、解决率、平均耗时

4. Scenario: 查看任务级别的轨迹分析
- **WHEN** 用户点击某个具体评测任务
- **THEN** 系统展示该任务的Agent执行轨迹，包括操作步骤序列、代码diff、执行耗时、token消耗

#### 异常场景

1. Scenario: 无评测数据时查看排行榜
- **WHEN** 系统中尚无任何评测结果
- **THEN** 排行榜页面展示空状态提示，引导用户触发首次评测

### 1.2 评测任务调度

#### 业务规则

1. Scenario: 创建评测任务
- **WHEN** 已认证用户选择模型、Agent、数据集并提交评测请求
- **THEN** 系统创建一条评测任务记录，状态为"排队中"，返回任务ID

2. Scenario: 评测任务执行
- **WHEN** 评测任务从队列中被取出执行
- **THEN** 系统在Docker沙箱中启动指定Agent，使用指定模型对数据集中的任务逐一执行评测，任务状态更新为"执行中"

3. Scenario: 评测任务完成
- **WHEN** 所有数据集任务执行完毕
- **THEN** 系统收集执行结果（通过/失败、轨迹、diff），计算解决率，更新任务状态为"已完成"，排行榜自动刷新

4. Scenario: 评测任务失败
- **WHEN** Agent执行过程中发生不可恢复的错误（如模型API不可达、沙箱异常）
- **THEN** 系统将任务状态更新为"失败"，记录错误信息，已完成的子任务结果予以保留

5. Scenario: 查看评测任务进度
- **WHEN** 用户查看正在执行的评测任务
- **THEN** 系统展示当前进度（已完成/总任务数）、当前正在执行的任务名称

6. Scenario: 同一模型+Agent+数据集组合的重复评测
- **WHEN** 用户对已评测过的同一组合再次发起评测
- **THEN** 系统创建新的评测任务，执行完成后排行榜展示最新结果，历史结果可查看

#### 异常场景

1. Scenario: 并发评测任务超过资源上限
- **WHEN** 同时执行的评测任务数超过配置的最大并发数
- **THEN** 新任务保持"排队中"状态，等待有执行槽位释放后再执行

2. Scenario: 单个任务执行超时
- **WHEN** 某个数据集子任务的Agent执行时间超过配置的超时阈值
- **THEN** 系统终止该子任务，标记为"超时"，继续执行下一个子任务

### 1.3 模型与Agent管理

#### 业务规则

1. Scenario: 注册新模型
- **WHEN** 已认证用户提交模型信息（名称、API端点、模型标识、类型-开源/闭源）
- **THEN** 系统注册该模型，使其可用于评测任务

2. Scenario: 注册新Agent
- **WHEN** 已认证用户提交Agent信息（名称、类型如Claude Code/OpenCode/CodeX、Docker镜像地址）
- **THEN** 系统注册该Agent，使其可用于评测任务

3. Scenario: 查看已注册模型列表
- **WHEN** 用户访问模型管理页面
- **THEN** 系统展示所有已注册模型的列表，包括名称、类型、状态

4. Scenario: 查看已注册Agent列表
- **WHEN** 用户访问Agent管理页面
- **THEN** 系统展示所有已注册Agent的列表，包括名称、类型、Docker镜像

#### 异常场景

1. Scenario: 注册模型时API端点不可达
- **WHEN** 用户提交模型信息时，系统验证API端点不可达
- **THEN** 系统提示验证失败，模型注册为"不可用"状态，仍可保存但不允许用于评测

### 1.4 数据集管理

#### 业务规则

1. Scenario: 查看可用数据集
- **WHEN** 用户访问数据集页面
- **THEN** 系统展示所有已配置的数据集，包括名称、语言、任务总数、描述、来源（内置/HuggingFace导入）

2. Scenario: 查看数据集任务列表
- **WHEN** 用户点击某个数据集
- **THEN** 系统展示该数据集包含的所有任务，包括任务ID、仓库名、Issue标题、难度等级

3. Scenario: 通过 HuggingFace 链接导入数据集
- **WHEN** 已认证用户提交一个 HuggingFace 数据集链接（如 `https://huggingface.co/datasets/princeton-nlp/SWE-bench`）
- **THEN** 系统在后台拉取该数据集的元信息和任务列表，解析完成后注册为可用数据集，期间展示"导入中"状态

4. Scenario: 一键同步内置主流数据集
- **WHEN** 已认证用户点击"同步内置数据集"
- **THEN** 系统自动录入以下预定义数据集（若已存在则跳过）：
  - `princeton-nlp/SWE-bench`（Python，300 tasks）
  - `princeton-nlp/SWE-bench_Lite`（Python，300 tasks 精简版）
  - `multi-swe-bench/multi-swe-bench`（多语言，含 Java/Python/JS 等）
  - `ByteDance-Seed/multi-swe-bench`（Java 子集，80 tasks）

5. Scenario: 查看数据集导入进度
- **WHEN** 用户在导入任务进行中查看数据集列表
- **THEN** 系统对"导入中"的数据集展示进度条或状态标签，导入完成后自动刷新

#### 异常场景

1. Scenario: 数据集配置文件缺失
- **WHEN** 系统启动时本地数据集配置文件不存在
- **THEN** 系统以空数据集列表启动，日志记录警告信息

2. Scenario: HuggingFace 链接无效或不可达
- **WHEN** 用户提交的链接解析失败（404、非数据集页面、网络超时）
- **THEN** 系统返回明确错误信息，数据集不被创建

3. Scenario: HuggingFace 数据集格式不兼容
- **WHEN** 拉取的数据集缺少必要字段（instance_id、repo、problem_statement）
- **THEN** 系统记录字段映射失败详情，将数据集标记为"解析失败"，任务数为 0

### 1.5 用户与权限

#### 业务规则

1. Scenario: 未认证用户查看排行榜
- **WHEN** 未认证用户访问排行榜、评测结果等只读页面
- **THEN** 系统正常展示内容，无需登录

2. Scenario: 已认证用户触发评测

- **THEN** 系统验证凭据，返回认证令牌

#### 异常场景

1. Scenario: 未认证用户尝试写操作
- **WHEN** 未认证用户尝试创建评测任务或注册模型
- **THEN** 系统返回401未授权错误，提示需要登录

## 数据约束变更

### ADDED

#### 2.1 评测任务

1. **任务ID**：系统自动生成的唯一标识符
2. **模型**：关联到已注册模型，不可为空
3. **Agent**：关联到已注册Agent，不可为空
4. **数据集**：关联到已配置数据集，不可为空
5. **状态**：枚举值，仅可为"排队中"、"执行中"、"已完成"、"失败"
6. **解决率**：0-100之间的数值，仅任务完成后有值
7. **创建时间**：任务创建的时间戳，不可为空
8. **完成时间**：任务完成的时间戳，仅任务完成后有值

#### 2.2 评测子任务结果

1. **所属任务ID**：关联到评测任务，不可为空
2. **数据集任务ID**：数据集中的任务标识符，不可为空
3. **结果**：枚举值，仅可为"通过"、"失败"、"超时"、"未执行"
4. **执行轨迹**：Agent操作步骤的JSON记录
5. **代码Diff**：Agent生成的代码变更差异
6. **执行耗时**：以秒为单位的执行时长
7. **Token消耗**：Agent执行消耗的token数量

#### 2.3 模型

1. **名称**：唯一，不可为空，最大长度128字符
2. **类型**：枚举值，仅可为"开源"、"闭源"
3. **API端点**：有效的URL格式
4. **模型标识**：用于API调用的模型标识符，不可为空
5. **状态**：枚举值，仅可为"可用"、"不可用"

#### 2.4 Agent

1. **名称**：唯一，不可为空，最大长度128字符
2. **类型**：枚举值，如"Claude Code"、"OpenCode"、"CodeX"等，可扩展
3. **Docker镜像**：有效的Docker镜像地址，不可为空
4. **状态**：枚举值，仅可为"可用"、"不可用"

#### 2.5 数据集

1. **名称**：唯一，不可为空，最大长度128字符
2. **语言**：数据集的编程语言，如"Java"、"Python"、"多语言"等
3. **任务总数**：非负整数（导入完成前为0）
4. **来源类型**：枚举值，仅可为"内置"、"huggingface"、"本地"
5. **HuggingFace repo ID**：当来源为 huggingface 时必填，格式为 `owner/dataset-name`
6. **导入状态**：枚举值，仅可为"就绪"、"导入中"、"解析失败"
7. **配置路径**：数据集配置文件的本地路径（来源为本地或内置时使用）

## 术语变更

### ADDED

**HuggingFace 数据集导入**
: 通过 HuggingFace Hub API 拉取公开数据集，解析其任务列表并注册为平台可用数据集的流程

**内置数据集（Builtin Dataset）**
: 平台预定义的主流评测数据集，含 HuggingFace repo ID 和字段映射规则，可一键同步，无需手动填写

### ORIGINAL ADDED

**解决率（Resolved Rate）**
: 在评测数据集中，Agent成功解决问题的任务数占总任务数的百分比

**AR2Code**
: Agent-Resolved-to-Code，指Agent自动解决真实GitHub Issue并生成代码修复的任务类型

**评测轨迹（Trajectory）**
: Agent在执行评测任务过程中的完整操作序列，包括工具调用、代码修改、中间输出等

**Docker沙箱**
: 用于隔离执行Agent评测任务的Docker容器环境，确保评测过程的安全性和可重复性
